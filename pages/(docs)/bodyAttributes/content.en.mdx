# `bodyAttributes`

The `bodyAttributes` setting adds attributes to the `<body>` tag.

This page is intentionally English-only so the new i18n flow has a real fallback case: `/bodyAttributes` and `/zh/bodyAttributes` both render, but the Chinese URL falls back to this file until a `content.zh.mdx` translation is added next to this route.

## Example

```ts
// pages/+config.ts

import type { Config } from 'vike/types'

export default {
  bodyAttributes: { class: 'dark' }
} satisfies Config
```

## Notes

- Use `bodyAttributes` for static body-level attributes.
- Use locale-aware content files when the prose itself must be translated.
- Add `pages/(docs)/bodyAttributes/content.zh.mdx` later when this page is ready for translation.
